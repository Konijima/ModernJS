# Internationalization (i18n)

ModernJS includes a built-in internationalization service to manage translations and locale switching.

## Configuration

Translations are typically defined in a configuration file (e.g., `i18n/config.js`).

```javascript
// i18n/en.js
export const en = {
    code: 'en',
    label: 'English',
    translations: {
        'HELLO': 'Hello, {0}!',
        'WELCOME': 'Welcome to ModernJS'
    }
};

// i18n/config.js
import { en } from './en.js';
import { fr } from './fr.js';

export const DEFAULT_LANGUAGE = 'en';
export const LANGUAGES = { en, fr };
```

## Usage

### Using the Pipe

The most common way to translate text is using the `translate` pipe in your templates.

```html
<h1>{{ 'WELCOME' | translate }}</h1>
<p>{{ 'HELLO' | translate : this.userName }}</p>
```

### Using the Service

You can also inject the `I18nService` to translate text programmatically or change the language.

```javascript
import { Component } from '../core/component/component.js';
import { I18nService } from '../core/services/i18n.service.js';

export class LanguageSwitcher extends Component {
    static inject = { i18n: I18nService };

    switchLanguage(langCode) {
        this.i18n.setLocale(langCode);
    }

    getGreeting() {
        return this.i18n.translate('HELLO', ['User']);
    }
}
```

## Reactivity

The `I18nService` is reactive. When the locale changes, all components using the `translate` pipe or subscribing to the service state will automatically update with the new translations.
